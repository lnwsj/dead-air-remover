Overview

ภาพรวมระบบ

Dead Air Remover เป็นเครื่องมือตัดช่วงเสียงเงียบ (dead air / silence) อัตโนมัติ เหมาะกับ podcast, วิดีโอ YouTube, voiceover, สัมภาษณ์ ทำงานด้วย Web Audio API + Canvas 2D ทั้งหมดในเบราว์เซอร์

F1 · Export state

Export state — ปุ่ม disable ขณะ encoding

เมื่อกด Export แล้ว ปุ่มทั้งหมด (Export, วิเคราะห์ใหม่, เปลี่ยนไฟล์) จะ disable อัตโนมัติ เพื่อป้องกัน race condition กับ state.abort

วิธีใช้

  1. อัปโหลดไฟล์ → รอวิเคราะห์
  2. กดปุ่ม Export → ปุ่ม disable ทันที
  3. รอจนเสร็จ → ปุ่มกลับมา enabled
ภาพที่ 12F1+F2: ปุ่ม Export disable + progress bar แสดง %
2
F2 · Progress bar

Progress bar — เห็นว่า export ไปถึงไหน

แถบ progress แสดง % ของการ export — ไฟล์ใหญ่ 70MB ใช้เวลา ~10s ผู้ใช้เห็น progress ตลอด

สูตรคำนวณ: video ~30s, audio ~5s — ถ้าเกิน 95% ระบบจะ complete เลย เพื่อหลีกเลี่ยงการค้างที่ 99%
3
F3 · Video export

WebM video export — ใช้ได้ทุกที่ยกเว้น Safari

ปัจจุบัน export วิดีโอเป็น WebM (VP9+Opus) ผ่าน MediaRecorder + captureStream + seek-based playback — เล็ก, เร็ว, เปิดได้บน Chrome/Edge/Firefox/Android

⚠️ ข้อจำกัด: WebM ไม่เล่นบน Safari/iPhone โดย default ถ้าต้องการ MP4 สำหรับ iPhone ให้ (1) ใช้ Chrome/Edge บน Mac แล้วแปลงด้วย HandBrake หรือ ffmpeg: ffmpeg -i input.webm -c:v libx264 -c:a aac output.mp4

วิธีใช้

  1. อัปโหลดไฟล์วิดีโอ (MP4/MOV/WebM)
  2. กดปุ่ม "Export" — ตรวจไฟล์อัตโนมัติ ถ้าเป็นวิดีโอจะ export เป็น WebM
  3. รอ progress 100% (แสดงขนาดไฟล์จริง) → ดาวน์โหลดอัตโนมัติ
🚀 v1.3 ที่จะมา: WebCodecs MP4 export (Chrome/Edge) + Auto-ffmpeg.wasm fallback จะมาใน release ถัดไป ตอนนี้ใช้ HandBrake แปลงได้ฟรี

F5 · Click-to-toggle

Click-to-toggle — คลิก waveform เพื่อ override

ถ้าแอปตัดผิด → คลิกที่ waveform เพื่อ toggle keep ↔ cut ทันที ไม่ต้องปรับ slider

F6 · A/B auto-switch

A/B auto-switch — เปรียบเทียบ original ↔ processed แบบสลับ

กด toggle เพื่อให้แอปเล่นสลับ original 5s → processed 5s → loop เปรียบเทียบง่ายๆ ว่าแอปตัดถูกจุดไหน

ภาพที่ 11F6: A/B toggle เปิด — เล่นสลับ original ↔ processed ทุก 5 วินาที

วิธีใช้

  1. อัปโหลดไฟล์ → วิเคราะห์เสร็จ
  2. กด toggle 🔄 A/B
  3. ฟัง 5s → เสียงจะตัด → กลับมาเล่น original
  4. เปรียบเทียบ: "ตรงนี้แอปตัดดีไหม? ตัดถูกจุด?"
  5. ปิด toggle เมื่อพอใจ → Export
ต้องมี processed buffer ก่อน: ถ้ายังไม่ได้วิเคราะห์ → จะขึ้นเตือน "วิเคราะห์ก่อน"

F8 · Adaptive threshold

Adaptive threshold — แอปคำนวณ noise floor อัตโนมัติ

แทนที่จะเดา threshold เอง → แอปจะสแกน 30th percentile ของ dB ทั้งไฟล์ แล้วตั้ง threshold = noiseFloor + 5dB

อัลกอริทึม

  1. สแกน RMS amplitude ทุก 10ms
  2. คำนวณ dB ของแต่ละ window
  3. เรียง dB แล้วหา 30th percentile (≈ noise floor)
  4. ตั้ง threshold = noiseFloor + 5dB (เก็บเฉพาะเสียงที่ดังกว่า noise ชัดเจน)
  5. วิเคราะห์ใหม่ทันที
ภาพที่ 8F8: กด Adaptive → แอปคำนวณ noise floor + 5dB อัตโนมัติ
เหมาะกับ: ไฟล์ outdoor, ไฟล์มี background noise, ไฟล์ที่ไม่รู้ noise level — แอปจะปรับให้อัตโนมัติ

F9 · Batch processing

Batch processing — ตัดทีละ 20 ไฟล์

ลากหลายไฟล์พร้อมกัน → แอปเพิ่มเข้า queue → กด Process all → แอปดาวน์โหลดไฟล์ที่ตัดเสร็จทีละไฟล์

วิธีใช้

  1. ลากหลายไฟล์ (Ctrl+Click) มาที่ drop zone
  2. แอปเพิ่มเข้า Batch Queue (แสดงทันที)
  3. เลือก preset ที่ต้องการ (Podcast/Video/Interview)
  4. กด ▶ Process all & download
  5. แอปจะ process ทีละไฟล์ → download แต่ละไฟล์เข้า Downloads folder
  6. เมื่อเสร็จจะแสดง "Batch: X/Y สำเร็จ"
ภาพที่ 13F9: Batch Queue แสดง 4 ไฟล์ พร้อม status (✓ done, ⚙ processing, ⏳ pending, ✗ error)
⚠️ ข้อจำกัด: ไฟล์ใหญ่ 50+ MB อาจใช้เวลานาน — แนะนำประมวลผล 5-10 ไฟล์ต่อครั้ง

F13 · Real export progress (v1.2)

Progress bar ที่ตามไบต์จริง

v1.1 progress เป็น time-estimate (คาดเดา) v1.2 progress = bytesWritten / expectedBytes — แม่นยำ 100%

อัลกอริทึม

  • Audio (WAV): encodeWAV() วัดเวลาจริง → 50% เมื่อ encode เสร็จ → 100% หลัง trigger download → toast แสดง sizeKB + ms
  • Video (WebM): MediaRecorder ondataavailable นับ bytes สะสม → tickExportProgress() ทุก chunk → เป้า = totalKeepDuration × 5.125 Mbps
  • Success toast: ดาวน์โหลด 5.6MB · ลด 92% จาก 70MB
ขนาดไฟล์ที่ export ได้: ขึ้นกับ codec (VP9 + Opus ใน WebM) + content (ยิ่งมี dead air มากยิ่งเล็ก) — ตัวอย่าง TikTok 70MB → WebM 5.6MB = ลด 92%

Workflows

Workflow สำหรับผู้ใช้ทั่วไป

🎬 Workflow A: ตัด video YouTube ที่อัดมา

  1. อัปโหลด MP4 → รอวิเคราะห์
  2. กด 🎬 Video preset (threshold = -35 dB)
  3. ดู waveform — ตรวจช่วง silence overlay
  4. เปิด 🔄 A/B เพื่อฟังเทียบ
  5. คลิก waveform ถ้ามี false positive (เช่น ช่วงเสียงเบาๆ ที่ไม่อยากตัด)
  6. กด Export WebM → ดาวน์โหลด

🎙️ Workflow B: ตัด podcast 30 นาที

  1. อัปโหลด MP3 → รอวิเคราะห์
  2. กด 🎙️ Podcast preset (-45 dB, 200 ms, 50 ms)
  3. ดู Segments list — ตรวจ false positive (เช่น หายใจยาว)
  4. คลิก waveform เพื่อ override
  5. กด Export WAV → ดาวน์โหลด

📦 Workflow C: ตัด 20 ไฟล์พร้อมกัน

  1. เลือก preset ที่ต้องการ (เช่น 🎬 Video)
  2. ลาก 20 ไฟล์มาที่ drop zone พร้อมกัน
  3. แอปเพิ่มเข้า Batch Queue
  4. กด ▶ Process all
  5. แอปดาวน์โหลด 20 ไฟล์เข้า Downloads folder

F11: Multi-File Tabs + Apply to All + Export All (v1.3)

เพิ่มใน v1.3: ทำงานหลายไฟล์พร้อมกันในหน้าเดียว ไม่ต้องรอ export ไฟล์นึงเสร็จแล้วค่อยโหลดอีกไฟล์

🎯 3 ของใหม่หลัก

  • Tab per file — คลิกสลับไปมาระหว่างไฟล์โดยไม่สูญเสีย state (segments, threshold, processed buffer)
  • Apply to All — เปลี่ยน threshold/min-silence/padding แล้วกดปุ่มเดียวให้ apply กับทุกไฟล์ที่ decode แล้ว
  • Export All — dropdown เมนู: เลือก export เฉพาะไฟล์นี้ หรือ export ทั้งหมดทีละไฟล์ (80ms delay ป้องกัน popup blocker)

📸 3 ไฟล์ loaded พร้อมกัน

ลากไฟล์ 3 ไฟล์ลง drop zone → 3 tabs ปรากฏด้านบน waveform → ไฟล์แรก auto-decode + analyze

⚙️ Apply to All

เปลี่ยน threshold จาก -40 dB → -55 dB → กด 📋 ใช้ค่านี้กับทั้งหมด → ไฟล์ทั้ง 3 ถูก re-analyze ด้วย threshold ใหม่ทันที (segments เปลี่ยน)

📦 Export All

กดปุ่ม ⬇ Export → dropdown เปิด → เลือก 📦 ทั้งหมด (3) → ดาวน์โหลด WAV ทั้ง 3 ไฟล์ตามลำดับ

🔄 สลับไฟล์ + ปิดไฟล์

คลิก tab → waveform + segments ของไฟล์นั้นแสดงทันที (ไม่ต้อง decode ใหม่ — cache ไว้ใน state.files[])

คลิก × บน active tab → ลบไฟล์ออกจาก queue → auto-switch ไป tab ข้างเคียง หรือ reset กลับ drop zone ถ้าเหลือไฟล์เดียว

🧠 State Architecture

state.files[] = [
  { id, file, name, kind, size,
    audioBuffer, segments, processedBuffer,
    duration, sampleRate, numChannels,
    status: 'pending' | 'decoded' | 'error',
    error: null | string },
  ...
]
state.activeFileIndex = -1 | 0..N-1

ทุกไฟล์เก็บ buffer/segments แยก → สลับไฟล์ได้ทันทีโดยไม่ต้อง decode ซ้ำ

FAQ

คำถามที่พบบ่อย

❓ ไฟล์ถูกส่งไปเซิร์ฟเวอร์ไหม?

ไม่ — ทุกอย่างทำในเบราว์เซอร์ ดู Network tab ใน DevTools ขณะอัปโหลดจะเห็นว่าไม่มี request ออกนอกเครื่อง

❓ ทำไมบางไฟล์ Export แล้วเป็น WebM ไม่ใช่ MP4?

WebM รองรับใน Chrome/Edge/Firefox แต่ Safari/iPhone ไม่รองรับ — ใช้ MP4 export (F3) ถ้าต้องเปิดบน iPhone

❓ threshold ค่าไหนเหมาะกับ podcast?

ใช้ Preset Podcast (-45 dB) เป็นค่า default — ถ้ายังตัดมากไป ลด threshold ลง (-50 dB) / ถ้าตัดน้อยไป เพิ่ม (-40 dB)

❓ ทำไม podcast ของผมไม่ถูกตัดเลย?

ลอง Adaptive (F8) — แอปจะคำนวณ noise floor ให้อัตโนมัติ หรือลด threshold ลงถ้าเสียงพูดเบา

❓ Batch ใหญ่ๆ จะใช้เวลาเท่าไหร่?
